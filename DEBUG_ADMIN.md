# Как найти ошибки в Strapi Admin Panel

## 1. Логи бэкенда (сервер)

На сервере смотрите, что пишет Strapi:

```bash
cd ~/tayga-development-website
docker compose logs -f backend
```

Откройте админку в браузере и смотрите, появляются ли в логах ошибки (красные строки или `error:`). Оставьте логи открытыми во время работы в админке.

---

## 2. Консоль и сеть в браузере

Откройте админку (например `http://ВАШ_IP:1337/admin`), затем:

1. **F12** → вкладка **Console**  
   Все красные сообщения — это ошибки. Скопируйте текст или сделайте скрин.

2. **F12** → вкладка **Network** (Сеть)  
   Обновите страницу (F5). Красные строки — неудачные запросы. Клик по такому запросу → вкладка **Response** или **Preview** — там часто видно текст ошибки от API.

По этим двум местам обычно видно точную причину (403, 404, 500, CORS, «Failed to fetch» и т.д.).

---

## 3. Проверка API с сервера

Убедитесь, что API вообще отвечает:

```bash
curl -s "http://localhost:1337/api/hero?populate=*"
```

Если пусто или ошибка — проблема в бэкенде или правах. Если приходит JSON — API работает, тогда ошибки скорее в админке (права админа, контент не опубликован и т.д.).

---

## 4. Типичные причины «ошибок и нет контента»

| Симптом | Что проверить |
|--------|----------------|
| 403 Forbidden в Network | **Settings** → **Users & Permissions** → **Public** → для нужных типов контента (Hero, Lead, …) включить **find** (и **findOne** при необходимости). Сохранить. |
| Пустой контент в админке | В **Content Manager** у записей должен быть статус **Published**. Черновики (Draft) не отдаются по API. |
| Ошибки при загрузке админки | В логах бэкенда и в Console/Network смотреть точный текст ошибки. Часто это неверный `APP_KEYS` / `ADMIN_JWT_SECRET` или несовпадение URL. |
| CORS / «blocked by CORS» | В `backend/.env` задать `FRONTEND_URL` (и при необходимости URL, с которого заходите в админку). В `config/middlewares.ts` уже есть CORS; при доступе по IP добавьте в `.env`: `FRONTEND_URL=http://ВАШ_IP:3000` и при необходимости разрешите свой домен/IP. |

---

## 5. Первый запуск (новая БД)

Если база была пустая и вы впервые заходите в админку:

- Strapi может показать форму **создания первого администратора**. Заполните и сохраните.
- После этого зайдите в **Content Manager**, создайте записи (например, Hero) и нажмите **Save & Publish**.

---

Итого: чтобы **найти ошибку**, смотрите по шагам 1 и 2 (логи бэкенда + Console и Network в браузере). По тексту ошибки или коду ответа (403/404/500) уже можно понять, что править (права, публикация контента, .env, CORS).
