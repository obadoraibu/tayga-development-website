# Отладка проблем с отображением данных из Strapi

## Проверка 1: Данные опубликованы в Strapi?

1. Откройте Strapi Admin: http://localhost:1337/admin
2. Перейдите в **Content Manager** → **Hero Section**
3. Убедитесь, что статус записи: **Published** (опубликовано)
4. Если статус **Draft**, нажмите **"Save & Publish"**

## Проверка 2: API работает?

Откройте в браузере:
```
http://localhost:1337/api/hero?populate=*
```

Должны увидеть JSON с данными. Если видите ошибку или пустой ответ - проблема в Strapi.

## Проверка 3: Права доступа

1. В Strapi Admin: **Settings** → **Users & Permissions Plugin** → **Roles** → **Public**
2. Найдите **Hero** в списке Permissions
3. Убедитесь, что галочка стоит на **find**
4. Сохраните

## Проверка 4: Кэш Next.js

Next.js кэширует данные. После изменений в Strapi:

1. **Остановите dev-сервер** (Ctrl+C)
2. Удалите папку `.next`:
   ```bash
   cd frontend
   rm -rf .next
   ```
3. **Перезапустите**:
   ```bash
   npm run dev
   ```

## Проверка 5: Консоль браузера

1. Откройте сайт в браузере
2. Нажмите F12 (DevTools)
3. Перейдите на вкладку **Console**
4. Проверьте, есть ли ошибки (красные сообщения)

## Проверка 6: Структура данных

Проверьте, что в Strapi поля заполнены правильно:
- **Title** - должен быть заполнен
- **Background Image** - должна быть загружена через Media Library
- Все поля должны быть сохранены и опубликованы

## Быстрое решение

Если ничего не помогает:

1. В Strapi: удалите запись Hero Section и создайте заново
2. Заполните все поля
3. Нажмите **"Save & Publish"**
4. Очистите кэш Next.js (удалите `.next`)
5. Перезапустите dev-сервер

## Проверка через терминал

Проверьте, что Strapi возвращает данные:

```bash
curl http://localhost:1337/api/hero?populate=*
```

Должен вернуться JSON с данными.
